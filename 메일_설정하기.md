# 사내메일로 브리핑 받기 — 설정 절차

메일 발송 기능은 **코드에 이미 들어 있습니다.** GitHub에 값 3개만 넣으면 켜집니다.
넣지 않으면 메일 단계는 그냥 건너뛰므로, 지금도 보드판은 정상 동작합니다.

> **비밀번호는 제(Claude)가 대신 넣어드릴 수 없습니다.**
> 아래는 직접 하셔야 하는 절차이고, 값은 저에게 알려주지 마세요.
> GitHub Secrets에 넣으면 저장소 관리자도 값을 다시 볼 수 없게 암호화됩니다.

---

## 0. 먼저 확인할 것 (이게 안 되면 나머지가 무의미)

**회사 메일이 외부에서 오는 메일을 받나요?**

회사 메일 주소로 개인 Gmail에서 테스트 메일을 한 통 보내보세요.

- **도착함** → 아래 절차를 진행하시면 됩니다.
- **안 옴 / 스팸함으로 감** → 사내 스팸필터에 막힌 겁니다.
  이 경우 메일 경로는 포기하고 **보드판 링크**를 쓰는 편이 낫습니다.
  (회사 IT에 화이트리스트 요청은 가능하지만 번거롭습니다)

> 참고: 회사에서 **Gmail 접속이 막혀 있어도** 메일 수신은 별개입니다.
> 막힌 건 '내가 Gmail 사이트에 들어가는 것'이고,
> 여기서 하려는 건 'Gmail이 회사 메일함으로 보내주는 것'입니다.

---

## 1. Gmail 앱 비밀번호 만들기

Gmail 계정 비밀번호를 그대로 쓰면 안 되고, **앱 전용 비밀번호**를 따로 발급받습니다.

1. Google 계정 → **보안** → **2단계 인증**을 먼저 켭니다. (안 켜면 앱 비밀번호 메뉴가 없습니다)
2. 같은 화면에서 **앱 비밀번호** 검색 → 새로 만들기
3. 이름은 아무거나 (예: `건설뉴스보드`)
4. **16자리 문자열**이 나옵니다. 이 창을 닫으면 다시 못 보니 바로 다음 단계로 가세요.

---

## 2. GitHub에 값 3개 등록

저장소 → **Settings** → 좌측 **Secrets and variables** → **Actions** → **New repository secret**

| 이름 | 값 | 설명 |
|---|---|---|
| `MAIL_USERNAME` | 보내는 Gmail 주소 | 예: `myid@gmail.com` |
| `MAIL_PASSWORD` | 1단계의 **16자리 앱 비밀번호** | Gmail 로그인 비밀번호가 **아닙니다** |
| `MAIL_TO` | **받을 회사 메일 주소** | 쉼표로 여러 명도 가능 |

이름은 **대소문자까지 정확히** 같아야 합니다. 하나라도 다르면 메일 단계가 조용히 건너뛰어집니다.

### 같이 넣으면 좋은 것 (선택)

**Settings → Secrets and variables → Actions → Variables** 탭 (Secrets 아님)

| 이름 | 값 |
|---|---|
| `BOARD_URL` | `https://dltjgus111-sys.github.io/HDEC_NEWS_BOARD/` |

넣으면 메일 하단에 **[보드판 전체 보기 →]** 버튼이 생깁니다.

---

## 3. 테스트

저장소 → **Actions** 탭 → 좌측 **건설 뉴스 보드 자동 갱신** → 우측 **Run workflow**

3~5분 뒤 메일이 도착하는지 확인하세요. (검색어가 53개라 수집에 3분쯤 걸립니다)

### 안 오면 확인 순서

1. **Actions 로그에서 `브리핑 메일 발송` 단계가 있는지**
   - 아예 없다 → `MAIL_USERNAME` 이름 오타. 이 값이 비면 단계 자체를 건너뜁니다.
   - 빨간 X → 로그를 열어보세요. 대개 앱 비밀번호 오류입니다.
2. **스팸함 확인** — 첫 메일은 스팸으로 가는 경우가 흔합니다. '스팸 아님' 처리해 두세요.
3. **회사 메일서버 거부** — 0번 항목으로 돌아가세요.

---

## 메일에 담기는 내용

- 오늘의 핵심 헤드라인 + 갱신 시각
- 지표 4칸 (유가 · 환율 · 기준금리 · 공사비지수)
- **6개 카테고리 전체 기사** (제목 클릭 → 원문, 언론사 · 교차보도 수 표시)
- 지난주 돌아보기
- 한 줄 코멘트 (관리자 화면에서 쓴 경우에만)
- 보드판 링크 버튼 (`BOARD_URL` 설정 시)

회사망에서 보드판이 막혀도 **메일 하나로 브리핑이 끝나도록** 전 기사를 담습니다.

---

## 발송 시각

메일은 **수집이 끝난 직후** 나갑니다. 즉 갱신 시각과 같습니다.

현재 예약: **04:47 / 06:23** (매일), **15:07 / 16:41** (평일)
→ 하루 최대 4통입니다. 많으면 워크플로에서 크론 줄을 지우면 됩니다.

> GitHub 예약 실행은 대기열에 밀릴 수 있어 시각이 다소 움직입니다.
> 그래서 아침·오후 각각 2번씩 걸어 하나가 밀려도 다른 하나가 들어오게 해뒀습니다.
