# Claude Design 인수인계 — 건설 뉴스 브리핑 보드

이 문서를 **Claude Design 대화창에 통째로 붙여넣고** 디자인을 요청하세요.
결과물은 `Share → Send to Claude Code`로 넘기면 제가 실제 보드에 이식합니다.

**현재 라이브**: https://dltjgus111-sys.github.io/HDEC_NEWS_BOARD/
디자인 요청 전에 이 화면을 캡처해서 함께 첨부하시면 맥락 전달이 훨씬 정확합니다.

---

## 0. 먼저 알아야 할 제약 (중요)

이 페이지는 **정적 목업이 아니라 매일 데이터가 바뀌는 화면**입니다.

- 카드 개수·글자 길이가 **매일 달라집니다.** 제목은 15자~60자까지 들쭉날쭉합니다.
- 기사 **사진이 없습니다.** 뉴스 수집 경로(Google News RSS)가 이미지를 제공하지 않습니다.
  → 사진 자리를 비워두는 디자인은 쓸 수 없습니다. 타이포·색·여백으로 위계를 만들어야 합니다.
- 최종 산출물은 **단일 HTML 1개 + CSS 인라인**입니다. 외부 라이브러리를 쓸 수 없습니다
  (회사망에서 CDN이 막힐 수 있어 자체 완결형이어야 합니다).
- 사용자는 **회사 PC 데스크톱 브라우저**에서 봅니다. 모바일은 부차적입니다.

> 그래서 요청하실 때 "이미지가 없는 뉴스 카드"라는 점을 꼭 언급해 주세요.

## 3. 디자인해 주셨으면 하는 것

우선순위 순입니다.

1. **뉴스 카드** — 이 화면의 핵심입니다.
   - 사진 없이 제목·언론사·날짜·신뢰도 배지만으로 위계를 만드는 방법
   - 제목 길이가 들쭉날쭉해도 카드 높이가 무너지지 않는 구조
   - 카테고리 6개를 색으로 구분하되 화면이 알록달록해지지 않게
2. **오늘의 핵심(헤드라인) 영역** — 눈길이 가장 먼저 닿는 곳. 후킹 문구를 어떻게 보여줄지
3. **신뢰도 배지** — `16개 언론사 보도` vs `단독`. 둘 다 부정적으로 보이지 않아야 함
   (단독이 나쁜 게 아니라 건설 뉴스 특성입니다)
4. **지표 4칸** — 상승 빨강 / 하락 파랑 (한국 금융 관례)
5. **오늘의 키워드 태그**

## 6. 요청 순서 (권장)

메모해 둔 워크플로우대로 **로우파이 먼저** 가는 편이 토큰도 아끼고 방향도 빨리 잡힙니다.

1. **와이어프레임**으로 배치만 먼저 — "6개 카테고리 카드를 한 화면에 넣는 최적 배치 3안"
2. 배치 확정 후 **디자인 시스템** — 컬러 토큰·타이포·카드 컴포넌트(기본/펼침/빈 상태)
3. **Publish 버튼 누르기** (안 누르면 프로토타입에서 못 씁니다)
4. **프로토타입**으로 하이파이 1장
5. `Share → Send to Claude Code` → 저에게 "이 디자인을 뉴스 보드에 적용해줘"

## 7. 검수 체크리스트

지난번 학습에서 실제로 문제가 됐던 항목들입니다.

- [ ] 제목이 40자 넘어갈 때 카드가 깨지지 않는가
- [ ] 기사가 0건인 카테고리의 빈 상태가 있는가
- [ ] 카드 펼침/접힘 상태가 둘 다 디자인돼 있는가
- [ ] 1920×1080에서 스크롤 없이 다 보이는가
- [ ] 반응형(노트북 1440px, 태블릿)에서 깨지지 않는가 — **가장 놓치기 쉬움**
- [ ] 인쇄(PDF)했을 때 읽을 만한가
- [ ] 6개 카테고리 색이 한 화면에서 산만하지 않은가
- [ ] 외부 폰트·아이콘 라이브러리에 의존하지 않는가
