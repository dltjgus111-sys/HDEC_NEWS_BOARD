## 0. 먼저 알아야 할 제약 (중요)

이 페이지는 **정적 목업이 아니라 매일 데이터가 바뀌는 화면**입니다.

- 카드 개수·글자 길이가 **매일 달라집니다.** 제목은 18자~60자까지 들쭉날쭉합니다.
- 기사 **사진이 없습니다.** 뉴스 수집 경로(Google News RSS)가 이미지를 제공하지 않습니다.
  → 사진 자리를 비워두는 디자인은 쓸 수 없습니다. 타이포·색·여백으로 위계를 만들어야 합니다.
- 최종 산출물은 **단일 HTML 1개 + CSS 인라인**입니다. 외부 라이브러리를 쓸 수 없습니다
  (회사망에서 CDN이 막힐 수 있어 자체 완결형이어야 합니다).
- 차트도 **직접 그린 SVG/CSS**입니다. Chart.js 같은 것을 쓸 수 없습니다.
- 사용자는 **회사 PC 데스크톱 브라우저**에서 봅니다. 모바일은 부차적입니다.

> 요청하실 때 "이미지가 없는 뉴스 카드", "라이브러리 없이 CSS/SVG만"을 꼭 언급해 주세요.

### ⚠️ 색은 함부로 바꾸면 안 됩니다

카테고리 6색은 **색약 검증(OKLab ΔE, Machado 2009 시뮬)을 통과한 조합**입니다.
원래 쓰던 경쟁사 크림슨(`#B91C4A`)이 수주 그린(`#008C46`)과 적록색약에서 ΔE 3.7로
구분이 안 돼(하한 6.0) 파랑으로 교체한 결과입니다.

- 통과 수치: 인접쌍 최저 **ΔE 14.6**(deutan) / 정상시야 최저 **28.1** / 흰 배경 대비 전원 **3:1 이상**
- **순서를 바꾸면 인접쌍이 달라져 검증이 무효**가 됩니다.
- 색을 바꾸고 싶으시면 바꾼 값을 알려주세요. 제가 검증기를 다시 돌려 통과 여부를 확인하겠습니다.

## 3. 디자인해 주셨으면 하는 것

우선순위 순입니다.

1. **뉴스 카드** — 이 화면의 핵심입니다.
   - 사진 없이 제목·언론사·날짜·신뢰도 배지만으로 위계를 만드는 방법
   - **카드 폭이 223px까지 좁아집니다**(6열일 때). 이 폭에서 제목이 읽히는 타이포
   - 제목 길이가 들쭉날쭉해도 카드 높이가 무너지지 않는 구조
   - 6개 카테고리 색을 유지하되 화면이 알록달록해지지 않게
2. **오늘의 핵심(헤드라인) 영역** — 눈길이 가장 먼저 닿는 곳. 후킹 배지를 어떻게 보여줄지
3. **신뢰도 배지** — `16개 언론사 보도` vs `단독`. **둘 다 부정적으로 보이지 않아야 합니다.**
   단독이 나쁜 게 아니라, 건설 뉴스는 원래 대부분 단독입니다(거시 뉴스만 10개사 넘게 씁니다)
4. **통계 화면의 차트 3종** — 가로막대 / 24시간 막대 / 선그래프.
   - 규칙: 모든 점에 숫자 라벨 금지(첫·끝·최댓값만), 축·그리드는 흐리게, 이중축 금지
   - 시간대 막대는 **단일 색**입니다(크기를 높이로 표현하므로 색으로 또 표현하지 않음)
5. **스크랩 목록 아이템** — 날짜별 묶음, 카테고리 색 표시
6. **지표 4칸** — 상승 빨강 / 하락 파랑 (한국 금융 관례)
7. **오늘의 키워드 태그**

## 6. 요청 순서 (권장)

**로우파이 먼저** 가는 편이 토큰도 아끼고 방향도 빨리 잡힙니다.

1. **와이어프레임**으로 배치만 먼저 — "6개 카테고리 카드를 한 화면에 넣는 최적 배치 3안"
2. 배치 확정 후 **디자인 시스템** — 컬러 토큰·타이포·카드 컴포넌트(기본/펼침/빈 상태)
3. **Publish 버튼 누르기** (안 누르면 프로토타입에서 못 씁니다)
4. **프로토타입**으로 하이파이 — 세 화면 각각
5. `Share → Send to Claude Code` → 저에게 "이 디자인을 뉴스 보드에 적용해줘"

## 7. 검수 체크리스트

지난 학습과 이번 제작에서 **실제로 문제가 됐던** 항목들입니다.

- [ ] 제목이 44자일 때 카드가 깨지지 않는가 (카드 폭 223px 기준)
- [ ] 기사가 0건인 카테고리의 빈 상태가 있는가
- [ ] 카드 펼침/접힘 상태가 둘 다 디자인돼 있는가
- [ ] **콘텐츠 높이가 920px를 넘지 않는가** — 1440×900에서도 확인
- [ ] 반응형(1440px, 1180px, 태블릿, 모바일)에서 깨지지 않는가 — **가장 놓치기 쉬움**
- [ ] 인쇄(PDF)했을 때 읽을 만한가 (접힌 기사도 펼쳐져 나와야 함)
- [ ] 6개 카테고리 색이 한 화면에서 산만하지 않은가
- [ ] **색을 바꿨다면 색약 검증을 다시 받았는가** (§0)
- [ ] 차트에 모든 점 숫자 라벨을 붙이지 않았는가
- [ ] 외부 폰트·아이콘·차트 라이브러리에 의존하지 않는가
- [ ] 관리자 모달(로그인/설정)도 디자인에 포함됐는가
