Claude를 켜두지 않아도 **매일 아침 자동으로** 보드판이 갱신되고 Gmail로 브리핑이 옵니다.
회사에서는 **보드판 주소만 열거나 Gmail만 확인**하면 됩니다.

```
   GitHub Actions (매일 07:30 KST)
        │
        ├─ Google News RSS 에서 뉴스 수집 ──┐
        ├─ Yahoo Finance 에서 유가·환율 ────┤
        │                                   ▼
        │                              data.js 생성
        │                                   │
        ├─ data.js 커밋 → GitHub Pages 보드판 갱신
        └─ Gmail 로 브리핑 메일 발송
```

수동으로 돌리고 싶으면 이 폴더에서:


기사가 0건이면 일부러 실패시키고 기존 `data.js` 를 지키니, 실패해도 보드판이 깨지지 않습니다.

## 1단계 — 저장소 만들고 올리기

GitHub에서 **New repository** → 이름 `HDEC_NEWS_BOARD` → **Public** 선택 → Create.

> **Public으로 하는 이유 2가지**
> 1. GitHub Pages는 무료 계정에서 Public 저장소만 지원합니다.
> 2. Public 저장소는 Actions 실행 시간이 **무제한 무료**입니다 (Private은 월 2,000분 한도).
>
> 공개되는 건 뉴스 수집 스크립트와 공개 뉴스 링크뿐이고, 비밀번호는 아래 Secrets에 암호화되어 들어가므로 공개되지 않습니다.

이 폴더에서 터미널을 열고:

```bash
git init
git add .
git commit -m "건설 뉴스 보드 최초 배포"
git branch -M main
git remote add origin https://github.com/dltjgus111-sys/HDEC_NEWS_BOARD.git
git push -u origin main
```

## 2단계 — GitHub Pages 켜기

저장소 **Settings → Pages** → Source를 **Deploy from a branch**, 브랜치는 **main / (root)** → Save.

1~2분 뒤 주소가 나옵니다:
`https://dltjgus111-sys.github.io/HDEC_NEWS_BOARD/`

**이 주소가 회사에서 여실 보드판입니다.** 즐겨찾기에 넣어두세요.

## 3단계 — Gmail 앱 비밀번호 만들기

일반 Gmail 비밀번호로는 SMTP 발송이 안 되고, **앱 비밀번호**가 따로 필요합니다.

1. <https://myaccount.google.com/security> 접속
2. **2단계 인증**이 꺼져 있으면 먼저 켜기
3. <https://myaccount.google.com/apppasswords> 접속
4. 앱 이름에 `건설뉴스보드` 입력 → 생성
5. 나오는 **16자리 비밀번호**를 복사 (공백은 빼고 붙여넣어도 됩니다)

> ⚠️ 이 16자리는 계정 비밀번호와 같은 위력을 가집니다.
> **저(Claude)를 포함해 누구에게도 알려주지 마세요.** 아래 4단계에서 직접 붙여넣으시면 됩니다.
> 유출됐다 싶으면 위 앱 비밀번호 페이지에서 즉시 삭제하면 무효화됩니다.

## 4단계 — Secrets 등록

저장소 **Settings → Secrets and variables → Actions → Secrets 탭 → New repository secret**

| Name | Value |
|---|---|
| `MAIL_USERNAME` | 보내는 Gmail 주소 (예: `xxx@gmail.com`) |
| `MAIL_PASSWORD` | 3단계에서 만든 **16자리 앱 비밀번호** |
| `MAIL_TO` | **받을 주소 — 회사 이메일을 넣으세요.** 여러 개면 콤마로 구분 |

> ### 💡 회사에서 Gmail이 안 열려도 괜찮습니다
> 메일을 발송하는 주체는 **GitHub 서버**입니다. 회사 PC는 Gmail에 접속하지 않습니다.
> Gmail 계정은 '발송 창구'로만 쓰이고, 메일은 `MAIL_TO`에 적은 **회사 메일함으로 도착**합니다.
> 그래서 `MAIL_TO`에는 반드시 회사에서 여는 메일 주소를 넣으세요.
>
> 메일 본문에 그날 브리핑 전체(핵심·지표 4종·뉴스·지난주·전망)가 담겨 있어서,
> 혹시 보드판 주소가 회사망에서 막히더라도 **메일만으로 완결**됩니다.

이어서 **Variables 탭 → New repository variable**

| Name | Value |
|---|---|
| `BOARD_URL` | `https://dltjgus111-sys.github.io/HDEC_NEWS_BOARD/` |

> 메일 설정을 안 하고 넘어가도 됩니다. `MAIL_USERNAME`이 없으면 메일 단계는 자동으로 건너뛰고
> 보드판 갱신만 됩니다.

### ② 메일이 도착하나

  IT팀에 외부 메일 차단 정책을 문의해 보세요.

### ③ `github.com`이 열리나 (선택)

`config.json`(기준금리·공사비지수·전망)을 회사에서 고치려면 필요합니다.
막혀 있어도 집에서 고치면 되니 필수는 아닙니다.

## 손이 가는 부분 (한 달에 한 번 정도)

`config.json`은 **GitHub 웹에서 연필 아이콘**으로 바로 고칠 수 있습니다. 회사에서도 됩니다.

- **기준금리 / 건설공사비지수** — 발표 주기가 길고 무료 API가 없어 수동입니다.
  금통위(연 8회)나 공사비지수 월간 발표 때 숫자만 바꿔주세요.
- **다음주 전망** — 판단이 들어가는 영역이라 자동 생성하지 않습니다. 주 1회 정도 고쳐주시면 됩니다.

유가·환율·뉴스는 전부 자동이라 손댈 일이 없습니다.

## 문제가 생기면

**메일이 안 옵니다**
Actions 탭에서 실행 로그의 "브리핑 메일 발송" 단계를 확인하세요.
`535 Authentication failed`면 앱 비밀번호가 틀렸거나 2단계 인증이 꺼진 경우입니다.
스팸함도 한 번 보세요.

**보드판이 어제 것으로 보입니다**
브라우저 캐시입니다. `Ctrl + F5`로 강제 새로고침하세요.
Actions가 초록색인데도 계속 그렇다면 커밋이 됐는지 저장소 기록을 확인해 보세요.

**`Permission denied` / `403` 으로 push 가 실패합니다** ← 가장 흔한 첫 실패

저장소가 기본값으로 **읽기 전용 권한**이라 `data.js` 커밋을 못 하는 경우입니다.

**Settings → Actions → General → Workflow permissions** →
**"Read and write permissions"** 선택 → Save.

그 다음 Actions 탭에서 Re-run 하면 됩니다.

**Actions가 빨간 X로 실패했습니다**
뉴스가 한 건도 안 걸리면 스크립트가 일부러 실패시킵니다 —
**기존 data.js를 빈 내용으로 덮어쓰지 않기 위해서**입니다. 보드판은 어제 것이 그대로 유지됩니다.
대개 일시적인 것이라 다음 날 정상화됩니다. 급하면 Run workflow로 다시 돌리세요.

**60일 넘게 저장소에 아무 활동이 없으면** GitHub이 스케줄을 자동 중지합니다.
매일 커밋이 일어나므로 실제로는 걸릴 일이 없습니다.

## 파일 구조

```
HDEC_NEWS_BOARD/
├── index.html                     보드판 화면 (자동 갱신이 건드리지 않음)
├── data.js                        뉴스 데이터 — Actions가 매일 덮어씀 (30일 보관)
├── config.json                    수동 관리 (언론사 목록·기준금리·공사비지수·전망)
├── 테스트실행.bat                  내 컴퓨터에서 즉시 테스트 (GitHub 불필요)
├── scripts/build_news.py          수집 스크립트
└── .github/workflows/daily-news.yml   자동 실행 설정
```

`index.html` 이 저장소 최상단에 있어야 Pages 주소가 바로 보드판으로 열립니다. **위치를 옮기지 마세요.**

## 뉴스 출처와 조정 방법

- **뉴스**: Google News RSS — 무료, API 키 불필요
- **유가**: Yahoo Finance 브렌트유 선물 (BZ=F)
- **환율**: Yahoo Finance USD/KRW, 실패 시 frankfurter.app(ECB)

| 바꾸고 싶은 것 | 고칠 곳 |
|---|---|
| 어느 언론사를 실을지 | `config.json` 의 `allowedPress` |
| 어떤 주제를 검색할지 | `build_news.py` 의 `CATEGORIES` → `queries` |
| 카테고리별 기사 수 | `build_news.py` 의 `CATEGORIES` → `max` |
| 며칠 이내 기사만 볼지 | `build_news.py` 의 `FRESH_DAYS` (기본 4일) |

`allowedPress` 는 **부분일치**라 `연합뉴스`만 적어도 `연합뉴스TV`가 함께 걸립니다.
목록을 좁힐수록 기사 수가 줄어드니, 고친 뒤엔 `테스트실행.bat` 으로 몇 건이 남는지 확인하세요.

## 카드에 기사 사진이 없는 이유

Google News RSS는 **기사 본문도 사진도 주지 않고** 제목·언론사·날짜·링크만 줍니다
(직접 확인했습니다). 그래서 카드 상단은 사진 대신 카테고리 그래픽으로 채웠습니다.

진짜 썸네일을 넣으려면 기사마다 원문 페이지를 열어 `og:image` 를 긁어와야 하는데,
구글뉴스 링크가 리다이렉트라 느리고 국내 언론사 상당수가 크롤러를 막아 실패율이 높습니다.
필요하시면 시도해 볼 수 있지만, 매일 안정적으로 도는 쪽을 우선했습니다.
